\# 电盾智检 —— 基于 Point-MAE 与多模态诊断的电力设施空间探伤系统



> 本项目参加中国高校计算机大赛-网络技术挑战赛 A 赛道（3D 点云方向），已通过资格赛，冲击省级奖项。



\## 一句话简介

针对电力巡检中二维视觉受限、缺陷样本少、落地门槛高等痛点，设计了一套“几何配准 + Point‑MAE 无监督学习 + VLM 多模态诊断”的全自动空间探伤系统。\*\*单次诊断耗时约 22 秒，探伤精度达到微米级\*\*。



\## 技术栈

\- \*\*语言\*\*：Python

\- \*\*深度学习\*\*：PyTorch（纯 PyTorch 实现，无 CUDA 强制依赖，CPU/GPU 兼容）

\- \*\*点云处理\*\*：Open3D、NumPy

\- \*\*大模型\*\*：Qwen‑VL 视觉语言模型（通过 API 调用）

\- \*\*前端界面\*\*：Gradio

\- \*\*配置管理\*\*：YAML



\## 我为什么做这个项目？



\- 传统二维巡检缺乏深度信息，易受光照、遮挡干扰，无法准确测量物理变形。

\- 有监督的 3D 检测需要大量缺陷标注，电力现场故障样本极少。

\- 工业部署环境复杂，算法常常因 CUDA 版本冲突无法运行。



我作为队长，主导了全部后端开发，并实现了\*\*零代码动态注册\*\*功能，让非专业人员也能快速接入新设备。

\## 主要技术亮点



\### 1. 无监督缺陷识别（不依赖标注样本）

\- 训练时只使用\*\*健康设备\*\*的点云，通过 60% 掩码率让 Point‑MAE 学习正常拓扑。

\- 推理时计算重建点云与输入点云的 Chamfer 距离，用 `mean + 2\*std` 动态阈值自动标记异常点。



\### 2. 全平台兼容的后端（解决 CUDA 依赖）

\- 用纯 PyTorch 重写了最远点采样（FPS）、KNN 搜索、Chamfer Distance 等算子。

\- \*\*无需 GPU，在普通笔记本上即可完成完整推理\*\*，适合工业现场部署。



\### 3. 零代码动态注册（自动化训练管线）

\- 上传一个标准模型 → 自动触发数据增强（缩放、旋转、高斯噪声）→ 微调训练 → 写入配置文件 → 前端热加载。

\- 全程无需修改代码，\*\*分钟级完成新设备接入\*\*。



\### 4. 多模态报告生成

\- 自动抓取异常区域的多视角 2D 截图，调用 Qwen‑VL 生成专业诊断报告（如“疑似鸟巢异物，建议清障”）。



\## 关键性能指标



\- \*\*端到端诊断耗时\*\*：≈ 22 秒（点云加载 1s + 目标提取 2s + MAE 推理 2s + 异常检测 1s + 多视角拍摄 3.25s + VLM 分析 15.75s）

\- \*\*异常检测召回率\*\*：> 85%

\- \*\*支持点云格式\*\*：.ply / .las / .txt

\- \*\*硬件兼容\*\*：纯 CPU 可运行
